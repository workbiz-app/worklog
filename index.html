📘 업무일지 간단 사용법

하루 5초면 익힙니다. 겁먹지 마세요!

1

오늘 할 일 추가

상단 입력창에 업무를 적고 Enter. 체크박스로 완료 표시, 우측 ▼ 해결방안/결과를 눌러 상세 메모와 사진을 첨부할 수 있습니다.

💡 Ctrl+V로 스크린샷 바로 붙여넣기 가능
2

하루 끝나면 "일과 종료"

완료 업무 = 왼쪽 📚 저장소로 자동 이동. 미완료 = 다음날로 이월 (＂어제 못 끝낸 업무＂ 배지 표시). 까먹어도 날짜 바뀌면 자동 저장됩니다.

3

보고서 다운로드

날짜 범위 선택 → PPTX(간결/상세/주간요약) · Excel · Word 다운로드. 주간/월간 성과보고에 바로 쓸 수 있어요.

4

오른쪽 패널 활용

🔗 링크: 자주 쓰는 사이트 북마크 · 📇 거래처: 국내/해외 업체 관리 · ✍ 문구: 복사용 템플릿 (클릭=복사)

5

지난 업무 검색

왼쪽 🔍 검색창에 키워드 입력. 과거 업무·메모에서 일치하는 기록이 모두 노란색으로 하이라이트되어 한 번에 보입니다.

6

다기기 싱크 (구글 로그인 시)

데스크탑에서 작성 → 핸드폰에서 열기 → 자동으로 같은 기록이 뜸. 출장 중에도 업무일지를 놓치지 않아요.

💡 게스트로 쓰다가 로그인하면 지금까지 기록이 자동 이관됩니다

🔧 첫 설치 가이드

한 번만 설정하면 모든 기기(핸드폰·노트북·데스크탑)에서 자동 싱크됩니다.
총 소요 시간 약 15분. 순서대로 따라오세요.

STEP 1 — GitHub Pages 무료 호스팅 (약 5분)

OAuth는 file://에서 작동 안 함 → 무료 웹 주소 필요.

  1. 가입: github.com/signup
  2. 저장소 생성: github.com/new — 이름 worklog, Public 선택
  3. 저장소에서 Add file → Upload files → 이 HTML 파일을 index.html로 이름 변경해서 업로드
  4. 저장소 Settings → Pages → Branch main / root → Save
  5. 1~2분 대기 → 주소 표시됨: https://[사용자명].github.io/worklog/
    ⭐ 이 주소 메모 (STEP 2에서 필요)
STEP 2 — Google OAuth Client ID 발급 (약 10분)

본인 Drive 접근 권한 — 1회만 발급.

  1. 접속: console.cloud.google.com (업무일지 저장할 구글 계정으로 로그인)
  2. 새 프로젝트 생성: 프로젝트 생성 (이름: 업무일지)
  3. Drive API 활성화: Drive API 페이지 → "사용" 클릭
  4. OAuth 동의 화면: 동의 화면 구성
    → User Type: 외부 → 앱 이름 업무일지, 이메일 본인 것 → 저장 후 계속
    → "테스트 사용자" 섹션에서 본인 Gmail 추가 (딱 본인만 추가하면 됨)
  5. Client ID 발급: 사용자 인증 정보
    → 상단 + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID
    → 유형: 웹 애플리케이션
    → 승인된 JavaScript 원본: https://[사용자명].github.io (주소 끝 /worklog 빼고 도메인까지만)
    → 생성 → Client ID 복사
STEP 3 — HTML에 Client ID 박기 (약 1분)
  1. index.html을 메모장으로 열기
  2. Ctrl+F로 여기에_CLIENT_ID_붙여넣기 검색
  3. 그 줄을 STEP 2에서 복사한 Client ID로 교체:
    const GOOGLE_CLIENT_ID = "1234-abc.apps.googleusercontent.com";
  4. 저장 → GitHub에 다시 업로드 (같은 파일명으로 덮어쓰기)
STEP 4 — 사용 시작 (딸깍 딸깍)
  1. GitHub Pages 주소로 접속 (핸드폰/노트북/데스크탑 아무거나)
  2. "구글로 로그인" 클릭
  3. 첫 1회만: "확인되지 않은 앱" 경고 → 고급 → 안전하지 않음(이동) (본인이 만든 앱이라 문제없음)
  4. 권한 허용 → ✅ 완료. 이후 자동 로그인
✅ 설정 후 사용법: 어떤 기기에서든 주소 접속 → 자동 로그인 → 바로 사용. 별도 로그인 버튼 클릭도 필요 없을 때가 많습니다 (이전 세션 자동 복원).
💡 v3 예정 기능: 팀장 초대코드로 팀원들과 공유 워크스페이스 사용 (별도 업데이트)
🔄 자동 로그인 해제: 브라우저 콘솔(F12)에서 localStorage.removeItem('drive_auto_login'); location.reload();
